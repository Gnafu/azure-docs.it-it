# <a name="workday-and-azure-active-directory"></a>Giorni lavorativi e Azure Active Directory

Connetti la giornata lavorativa con Azure Active Directory (Azure AD) usando l'integrazione predefinita ed effettuando automaticamente il provisioning o il deprovisioning degli account utente quando i dipendenti si uniscono, si spostano o abbandonano.

![Mostra il logo dell'app giorni lavorativi](./media/workday.png)

> [!div class="nextstepaction"]
> [Prova la giornata lavorativa con Azure AD >](https://aka.ms/WorkdayTryItNow)

## <a name="what-can-you-do-with-workday-and-azure-ad"></a>Cosa puoi fare con la giornata lavorativa e Azure AD?

- Automatizzare il provisioning **degli utenti** : assegnare automaticamente le applicazioni ai dipendenti in base al gruppo, al ruolo o ad altri attributi nella giornata lavorativa.
- **Sincronizzare i profili dei dipendenti** : assicurarsi che i dati dei dipendenti rimangano aggiornati tra Azure ad e giorni lavorativi.
- **Semplifica** l'onboarding e il Offboarding: effettua automaticamente il provisioning degli account utente per i nuovi assunti, in modo che possano essere produttivi il giorno uno e ridurre i rischi per la sicurezza tramite il deprovisioning automatico degli account utente quando i dipendenti vengono lasciati.
- **Easy Single Sign-on** Eliminare nome utente e password e consentire agli utenti di accedere usando i propri account aziendali ospitati in Azure AD.

## <a name="get-started"></a>Attività iniziali

Configurare il [provisioning degli utenti](https://aka.ms/WorkdayProvisioning) e l' [accesso Single Sign-on](https://aka.ms/WorkdaySSODoc) per la giornata lavorativa con le semplici guide alla configurazione.
