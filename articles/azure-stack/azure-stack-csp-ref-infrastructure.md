## <a name="usage-reporting-infrastructure-for-cloud-service-providers"></a>Utilizzo di reporting dell'infrastruttura per il provider di servizi Cloud

Stack di Azure include l'infrastruttura necessaria per tenere traccia dell'utilizzo, poiché si verifica e lo inoltra a Azure. In Azure, Azure Commerce elabora i dati di utilizzo e gli addebiti di utilizzo per le sottoscrizioni di Azure appropriate, esattamente come avviene nel cloud di Azure globale di utilizzo.

È necessario tenere presente che alcuni concetti sono coerenti tra globale Azure e Azure Stack. Stack di Azure dispone di sottoscrizioni locali, che soddisfano un ruolo simile a una sottoscrizione di Azure. Sottoscrizioni locali sono valide solo localmente. Sottoscrizioni locali vengono mappate alle sottoscrizioni di Azure quando viene inoltrato utilizzo in Azure.

Stack di Azure dispone metri utilizzo locale. Utilizzo locale viene eseguito il mapping per i controlli utilizzati in Azure commerce. Tuttavia, gli ID misuratore sono diversi. Sono disponibili più misuratori in locale rispetto a quello utilizzato da Microsoft per la fatturazione.

Esistono alcune differenze tra il prezzo servizi nello Stack di Azure e Azure. Ad esempio, nello Stack di Azure, l'addebito per le macchine virtuali è basato solo su vcore/ore, con lo stesso importo per tutte le serie di macchina virtuale, a differenza di Azure. Il motivo è che in Azure globale prezzi diversi riflettono un hardware diverso. Nello Stack di Azure, il cliente fornisce l'hardware, pertanto non c'è alcun motivo per addebitare ritmi diversi per diverse classi di macchina virtuale.

È possibile scoprire i controlli di Stack Azure utilizzati in commercio e i relativi prezzi Partner Center, esattamente come si farebbe per servizi di Azure:

1. In Partner Center, andare al **menu del Dashboard** > **prezzi e offerte speciali**.
2. In **basata sull'utilizzo di servizi**selezionare **corrente**.
3. Aprire il **Azure nell'elenco prezzi CSP globale** foglio di calcolo.
4. Filtrare in base a **area = Stack Azure**.

## <a name="terms-used-for-billing-and-usage"></a>Termini utilizzati per la fatturazione e utilizzo

I seguenti concetti e termini vengono utilizzati per l'utilizzo e fatturazione nello Stack di Azure:

| Periodo | Definizione |
| --- | --- |
| Partner CSP diretto | Un partner diretto di Provider di soluzioni Cloud (CSP) riceve una fattura direttamente da Microsoft per l'utilizzo di Azure e Azure Stack e i clienti di pagamenti direttamente. |
| CSP indiretta | Rivenditori indiretti utilizzano un provider indiretto (noto anche come server di distribuzione). La selezione di rivenditori clienti finali. il provider indiretto contiene la relazione di fatturazione con Microsoft gestisce fatturazione del cliente e fornisce servizi aggiuntivi, ad esempio il supporto tecnico. |
| Utenti finali | I clienti di fine sono di aziende ed enti pubblici che possiedono le applicazioni e altri carichi di lavoro nello Stack di Azure. |
