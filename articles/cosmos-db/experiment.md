# <a name="azure-cosmos-db-documentation-experiment"></a>Esperimento sulla documentazione di Azure Cosmos DB

Azure Cosmos DB è un database multimodello distribuito a livello globale. Dato che con Azure Cosmos DB è possibile eseguire numerose operazioni, dalla creazione di grafi alla creazione di database di documenti, fino alla migrazione dei dati dalle risorse di archiviazione MongoDB, Cassandra o Tabella di Azure, organizzare la documentazione può rivelarsi un'operazione complessa. A tal proposito, Microsoft chiede gentilmente la collaborazione degli utenti.

Con riferimento al test in corso sull'organizzazione del sommario e della [pagina di destinazione](https://docs.microsoft.com/en-us/azure/cosmos-db/) di Azure Cosmos DB, come illustrato nelle immagini seguenti. Microsoft sarà lieta di ricevere commenti e suggerimenti sulle modifiche proposte.

## <a name="to-load-both-views"></a>Per caricare entrambe le visualizzazioni

1. Per aprire le due visualizzazioni affiancate, passare alla [pagina di destinazione](https://docs.microsoft.com/en-us/azure/cosmos-db/) dal Web browser preferito. 
2. Fare riferimento alle immagini precedenti per stabilire qual è la versione visualizzata, quella originale o quella sperimentale. 
3. Aprire una finestra del browser InPrivate o in incognito, caricare la [pagina di destinazione](https://docs.microsoft.com/en-us/azure/cosmos-db/) nella nuova finestra del browser e fare nuovamente riferimento alle immagini per stabilire qual è la versione visualizzata. 

   Se non si hanno di fronte entrambe le visualizzazioni, chiudere e riaprire una nuova finestra InPrivate o in incognito e caricare nuovamente la [pagina di destinazione](https://docs.microsoft.com/en-us/azure/cosmos-db/). Per caricare entrambe le versioni in finestre separate possono essere necessari diversi tentativi. Con l'aggiornamento della finestra in incognito non viene caricata l'altra versione, ma è necessario aprire una nuova finestra. 

4. È quindi possibile ridimensionare le finestre e affiancarle o passare da una visualizzazione all'altra per confrontarle. 

5. Dopo aver acquisito familiarità con l'organizzazione e il layout della versione sperimentale, fare clic sul pulsante **Experiment survey** (Sondaggio sull'esperimento) di seguito per fornire eventuali commenti. Inviare il modulo entro la mezzanotte del 4 aprile 2018 per consentire di esaminare e integrare i commenti.
