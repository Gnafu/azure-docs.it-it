# <a name="comparing-appsource-and-azure-marketplace"></a>Confronto tra AppSource e Azure Marketplace  
Le vetrine soddisfano i requisiti univoci del cliente. Le vetrine consentono il scegliere i destinatari in base al ruolo. Potrebbe essere possibile offrire la soluzione o il servizio più adeguato in base al cliente.  

## <a name="understanding-the-differences-between-the-storefronts"></a>Differenze tra le vetrine  
Quando si sceglie una vetrina è necessario prima di tutto identificare i destinatari della propria offerta. In caso di entrambi i destinatari, è possibile pubblicare una volta per vendere in entrambe le vetrine.  
 
Prendere in considerazione i vantaggi aggiuntivi per le vetrine.   

### <a name="appsource"></a>AppSource  

| Vantaggi | Dettagli | 
|:--- |:--- | 
| ***Flessibilità nella fatturazione*** | Esegue il provisioning di un'esperienza di valutazione. Attualmente non offre un'opzione di pubblicazione abilitata per la commercializzazione. Ciò consente di usare l'attuale infrastruttura per gli ordini e la fatturazione senza che siano necessari altri investimenti o modifiche. | 
| ***Facilità di collegamento con altri partner*** | Collega fornitori di software indipendenti, integratori di sistemi e provider di servizi gestiti a specifici scenari di implementazione. È possibile vendere in modalità collaborativa a nuovi clienti. | 
| ***Automazione*** | Collega fornitori di software indipendenti, integratori di sistemi e provider di servizi gestiti a specifici scenari di implementazione. È possibile vendere in modalità collaborativa a nuovi clienti. |  
| ***Più tipi di cloud*** | Non supporta attualmente Azure Stack, Azure per enti pubblici o cloud internazionali. | 
| ***Presentazione contestualizzata ai clienti*** | Consente di raggiungere più clienti tramite l'esperienza in-app per prodotti Microsoft, ad esempio Dynamics 365, Power BI e Office 365. |  

### <a name="azure-marketplace"></a>Azure Marketplace  

| Vantaggi | Dettagli | 
|:--- |:--- | 
| ***Flessibilità nella fatturazione*** | Per le opzioni di fatturazione con pagamento in base al consumo delle macchine virtuali vengono usati i contratti Enterprise Microsoft o i modelli di vendita diretta sul Web. Le opzioni di prezzo includono una sottoscrizione di livello gratuito, che prevede un'offerta perennemente gratuita. Le opzioni di prezzo includono inoltre una sottoscrizione da provare subito. Una sottoscrizione per una versione di valutazione (disponibile subito) offerta gratuitamente a fini promozionali per un periodo di tempo limitato, al termine del quale la sottoscrizione diventa a pagamento. L'attivazione BYOL rappresenta un'altra opzione di supporto per l'utente.<br />Per entrambe le opzioni di fatturazione e le offerte per macchine virtuali seguenti, tutte le risorse di Azure con provisioning vengono fatturate direttamente al cliente.<ul> <li>App di Azure: App gestita</li> <li>App di Azure: Modello di soluzione</li> </ul> | 
| ***Facilità di collegamento con altri partner*** | Non si è attualmente in grado di collegare un provider di servizi o un partner di consegna all'offerta. Questa funzionalità verrà avviata nel 2018. | 
| ***Automazione*** | Non si è attualmente in grado di collegare un provider di servizi o un partner di consegna all'offerta. È possibile trarre vantaggio dal provisioning automatizzato di soluzioni SaaS con componenti aggiuntivi. Usare le app di Azure: Modello di soluzione per automatizzare gli scenari di distribuzione e raccolta di dati basati su SaaS. |  
| ***Più tipi di cloud*** | È possibile pubblicare nel cloud pubblico e su soluzioni locali usando i tipi seguenti.<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>cloud regionali, tra cui Cina e in Germania</li> </ul> | 
| ***Presentazione contestualizzata ai clienti*** | È possibile rendere disponibile la propria soluzione all'interno del portale di Azure per ricerche contestuali mediante macchine virtuali e le app di Azure: Modello di soluzione. |
