# <a name="ibm-workloads-on-azure"></a>Carichi di lavoro di IBM in Azure

Molti carichi di mainframe IBM z/OS in base possono essere replicate in Azure senza alcuna perdita di funzionalità e senza notando anche le modifiche nel sistema sottostante per gli utenti. Rehosting di applicazioni in Azure ti offre funzionalità simili a mainframe che necessarie oltre l'elasticità, la disponibilità e i potenziali risparmi del cloud.

Azure supporta l'integrazione con gli ambienti mainframe IBM esistenti, consentendo di eseguire la migrazione di applicates significativi, eseguire soluzioni ibride in cui è necessaria e la migrazione nel corso del tempo. Anche se è possibile riscrivere completamente i programmi esistenti basati su mainframe per Azure, è più comune di rehost li. Riscrittura aggiunge costi, complessità e il tempo per i progetti di migrazione. Con rehosting, è possibile:

- Spostare le applicazioni in un emulatore basato su cloud.

- Eseguire la migrazione del database a un database basato sul cloud.

- Sostituire i moduli e del codice con motori di trasformazione di codice.

Inoltre, il software IBM, tra cui WebSphere ed MQ, è ora in Azure Marketplace. Con una licenza per il software IBM, puoi usufruire della scalabilità dell'infrastruttura on demand offerta da Azure per avviare rapidamente una macchina virtuale.

Un ecosistema di partner completo è disponibile per eseguire la migrazione di sistemi mainframe IBM in Azure. La maggior parte seguire un approccio pragmatico riutilizzo ove possibile prima di avventurarsi in una distribuzione graduale di riscrittura o la sostituzione delle applicazioni. Ottenere informazioni aggiuntive e aiutare i partner alla [centro migrazione Mainframe Azure](https://azure.microsoft.com/migration/mainframe/).
